HTTP/2

The next generation of the protocol that powers the open web. The Necko Team, Mozila Summit Innovation Fair, 5 October 2013

What does it do?

What does it do?

  • Unlimited parallelism!
  • Prioritization
  • Scalablity
  • Always Secure
  • Server Push
Unlimited parallelism - no more 6 resources per server limitation. No more sharding.
Prioritization - don't care about cat pictures? Tell the server to send dog pictures first!
Scalability - connection management prevents using lots of resources.
Always secure - we're committed to not running HTTP/2 unless it's over TLS.
Server push - "Users who downloaded index.html also downloaded...".

Why?

Why do we want to do this? Doesn't HTTP/1 work great already?

Why?

  • Performance (especially for mobile!)
  • One object = One URI
  • Promote Security
  • Promote Scalability
  • Play nice with others
Image by Chris Heilmann
We want to improve performance for our users, especially in the mobile space.
We want to promote the URI-per-object model of the web.
Promoting security and scalability protects the future of the web.
The connection management in HTTP/2 plays much nicer with the internet at large than HTTP/1 does.

Status

  • SPDY/2, SPDY/3, SPDY/3.1, HTTP/2-draft04, HTTP/2-draft06 — All contributed to the community.
  • Facebook, Twitter, Google, Wordpress, NginX1, Apache2 — All have one of the above available today.
  • Firefox, Chrome(ium), Opera, IE11 — All can speak SPDY/3 (or higher).
  • Standardization expected within a year.
